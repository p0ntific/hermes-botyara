# Hermes Botyara 🚀

AI-агент (Telegram Userbot) для автоматического поиска и квалификации B2B лидов в Telegram-чатах. Работает на пуле аккаунтов с общей очередью лидов и подключается к любой LLM через OpenAI-совместимый протокол (OpenRouter, Yandex Cloud, OpenAI, свой vLLM/Ollama).

## Функционал
1. Сканирует целевые чаты (супергруппы) и складывает лидов в общую персистентную очередь.
2. Диспетчер раздает лидов свободным аккаунтам: учитываются дневные лимиты, кулдауны и здоровье каждого аккаунта.
3. Анализирует каждого лида через LLM, отсеивает спам и саморекламу (фильтр `SKIP`).
4. Формирует персонализированный питч (с паузами для имитации человека).
5. Ведет диалог в личных сообщениях до выявления интереса (классификатор стадий + генератор ответов).
6. Отправляет теплых лидов менеджеру, пишет полный транскрипт каждого диалога и лог всех LLM-вызовов.

## Архитектура

```
TARGET_CHAT (лиды от сканера)
        │  все аккаунты слушают, enqueue идемпотентен
        ▼
┌─────────────────────┐
│  SQLite (hermes.db) │  очередь лидов, статусы, транскрипты,
│                     │  кулдауны аккаунтов, лог LLM-вызовов
└─────────┬───────────┘
          ▼
     Dispatcher ──── выбирает аккаунт: жив, без кулдауна, лимит не исчерпан,
          │          дольше всех не отправлял (LRU)
          ▼
 AccountWorker × N ── питч + свои диалоги (ответы приходят на тот же аккаунт)
          │
          ▼
     SalesBrain ──── LLMRouter: pitch / classify / reply могут ходить
                     в разные модели; fallback-провайдер при падении основного
```

- **Отказоустойчивость**: очередь и статусы переживают рестарт (зависшие в обработке лиды возвращаются в очередь); FloodWait затрагивает только один аккаунт — остальные продолжают; упавший воркер перезапускается с backoff; аккаунт с убитой сессией отключается с уведомлением менеджеру, не задевая остальных; при падении LLM-провайдера запросы уходят в fallback.
- **Прозрачность**: каждый входящий/исходящий месседж и каждое решение классификатора (стадия, confidence, модель) пишутся в SQLite; смотрится через CLI (ниже).

## Гарантии доставки лидов
- Согласившийся лид никогда не теряется: handoff обрабатывается даже при исчерпанном лимите ответов, после статуса «ручная проверка» и при ошибке отправки сообщения клиенту (менеджер получает пометку «напишите сами»).
- Все уведомления менеджеру идут через SQLite-outbox `admin_notifications`: запись удаляется только после подтверждённой доставки, а при сбое повторяется каждые `NOTIFY_RETRY_INTERVAL_SECONDS` и после рестарта.
- Кулдаун, FloodWait и дневной лимит одного аккаунта возвращают лида в общую очередь для другого доступного аккаунта. После исчерпания попыток лид получает `manual_required`, а менеджер — уведомление с готовым текстом.

## Быстрый старт (через Docker)

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/p0ntific/hermes-botyara.git
   cd hermes-botyara
   ```
2. Скопируйте конфиги и заполните свои данные:
   ```bash
   cp .env.example .env
   cp accounts.example.yaml accounts.yaml
   nano .env
   ```
3. Запустите бота:
   ```bash
   docker-compose up -d
   ```

Старый `mtproto_leads.json` при первом запуске автоматически импортируется в SQLite — история контактов не теряется. Legacy-режим одного аккаунта через `API_ID`/`API_HASH`/`SESSION` в `.env` продолжает работать без accounts.yaml.

## Как добавить Telegram-аккаунт

Добавьте блок в `accounts.yaml` (секреты можно выносить в `.env` через `${VAR}`) и перезапустите:

```yaml
accounts:
  - name: main
    api_id: ${API_ID}
    api_hash: ${API_HASH}
    session: ${SESSION}
  - name: second
    api_id: ${API_ID_SECOND}
    api_hash: ${API_HASH_SECOND}
    session: ${SESSION_SECOND}
    proxy: socks5h://127.0.0.1:9052   # свой прокси на аккаунт
    cold_dm_daily_limit: 3            # свой дневной лимит
```

Диалог всегда продолжает тот аккаунт, который писал лиду первым.

## Как переключать LLM

Все провайдеры говорят по OpenAI-совместимому протоколу, поэтому смена модели — это правка `.env`, без кода:

```bash
LLM_PROVIDER=openrouter
OPENROUTER_API_KEY=sk-or-...
OPENROUTER_MODEL=anthropic/claude-sonnet-4.5   # дефолт

# разные модели на разные стадии пайплайна:
LLM_MODEL_CLASSIFY=anthropic/claude-haiku-4.5  # классификатор можно удешевить
LLM_MODEL_PITCH=anthropic/claude-sonnet-4.5    # питч оставить на сильной модели

# страховка: если OpenRouter лег, вызовы уходят в Yandex
LLM_FALLBACK_PROVIDER=yandex
YANDEX_CLOUD_API_KEY=...
YANDEX_CLOUD_FOLDER=...
```

Дефолтная модель — `anthropic/claude-sonnet-4.5`: сильный русский язык, точное следование инструкциям промпта и стабильный JSON в классификаторе. Для экспериментов на openrouter.ai/models можно смотреть `anthropic/claude-haiku-4.5` (дешевле, для classify), `deepseek/*` и `google/gemini-*` — качество/латентность каждого вызова видно в `llm`-логе (ниже), так что сравнение моделей сводится к правке `.env` и просмотру лога.

Провайдеры: `openrouter`, `yandex`, `openai`, `custom` (любой OpenAI-совместимый сервер: `LLM_BASE_URL` + `LLM_MODEL`).

## Прозрачность диалогов

```bash
python -m hermes.transcript leads                 # все лиды: статус, стадия, аккаунт
python -m hermes.transcript show ivan_lead        # полный транскрипт диалога + решения ИИ
python -m hermes.transcript queue                 # состояние общей очереди
python -m hermes.transcript llm                   # последние LLM-вызовы: модель, латентность, ошибки
```

В Docker: `docker-compose exec hermes-bot python -m hermes.transcript leads`.

## Настройка GitHub Actions для деплоя
В репозитории настроен автоматический деплой на ваш сервер при пуше в ветку `main`.
Для этого добавьте в настройки репозитория на GitHub (Settings -> Secrets and variables -> Actions) следующие секреты:
- `SERVER_HOST`: IP адрес вашего сервера
- `SERVER_USER`: Имя пользователя SSH
- `SERVER_SSH_KEY`: Приватный SSH ключ (Ed25519 или RSA)

*Скрипт деплоя копирует `hermes/`, `tg_lead_skill.py` и `requirements.txt` в `~/.hermes` и перезапускает `tg_lead_skill.service` (точка входа не менялась: `python tg_lead_skill.py`). Не забудьте положить на сервер `accounts.yaml` рядом с кодом или оставить legacy-переменные в `.env`.*
